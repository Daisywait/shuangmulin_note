## 🚦 状态码概览

HTTP 状态码是服务器对客户端（浏览器）请求的响应“暗号”，由三位数字组成。

|   |   |   |   |
|---|---|---|---|
|**类别**|**范围**|**核心含义**|**简单记忆**|
|**1xx**|100-199|**信息提示**|"收到了，正在处理，请稍候..."|
|**2xx**|200-299|**请求成功**|"成功！这是你要的数据。"|
|**3xx**|300-399|**重定向**|"资源换地方了，你去那边找。"|
|**4xx**|400-499|**客户端错误**|"是你弄错了（请求有问题）。"|
|**5xx**|500-599|**服务器错误**|"是我搞砸了（服务器内部故障）。"|

## ✅ 2xx：请求成功 (Success)

这一类状态码表示服务器成功接收、理解并处理了请求。

### 1. `200 OK`

- **含义**：请求成功，通常返回网页内容或数据。
    
- **场景**：正常打开一个网页。
    

### 2. `201 Created`

- **含义**：请求成功，并且服务器**创建了新的资源**。
    
- **场景**：注册新用户、上传图片、发布新文章。
    

### 3. `204 No Content`

- **含义**：服务器处理成功，但**没有内容**需要返回。
    
- **场景**：点击“删除”按钮，删除成功后页面不需要刷新或显示新内容。
    

### 4. `206 Partial Content` (重点难点)

- **含义**：服务器只返回了请求资源的**一部分**。
    
- **机制**：客户端在请求头中发送 `Range` 字段（例如 `Range: bytes=500-999`），服务器响应此范围的数据。
    
- **场景**：
    
    - **断点续传**：大文件下载中断后，从断开的地方继续下载。
        
    - **视频流媒体**：视频播放器只缓冲当前播放进度的片段，而不是一次性加载整部电影。
        

## ↪️ 3xx：重定向 (Redirection)

这一类状态码表示客户端需要采取进一步的操作（通常是跳转到新 URL）才能完成请求。

### 1. `301 Moved Permanently` (永久重定向)

- **含义**：资源已被**永久**移动到新位置。
    
- **作用**：搜索引擎会更新索引，将旧网址的权重转移到新网址（对 SEO 非常重要）。
    
- **类比**：搬家后通知邮局和朋友这一辈子都住新地址了。
    

### 2. `302 Found` (临时重定向)

- **含义**：资源**暂时**移动到了新位置。
    
- **作用**：客户端这次去新地址，但下次还是访问旧地址。
    
- **类比**：房子装修，暂时去住酒店几天。
    

### 3. `304 Not Modified` (未修改 - 缓存机制)

- **含义**：资源未修改，可以直接使用**本地缓存**。
    
- **场景**：浏览器问服务器：“图片改了吗？”服务器说：“没改（304）”。浏览器直接从硬盘加载图片，**节省带宽，加载更快**。
    

## 🚫 4xx：客户端错误 (Client Error)

这一类状态码表示**客户端**发送的请求有误，服务器无法处理。

### 1. `400 Bad Request`

- **含义**：请求语法错误，服务器读不懂。
    
- **场景**：参数格式不对，或者请求体数据损坏。
    

### 2. `401 Unauthorized` (未授权)

- **含义**：请求需要身份验证（没有登录或证书无效）。
    
- **类比**：去图书馆没带借书证。
    

### 3. `403 Forbidden` (禁止)

- **含义**：服务器理解请求，但**拒绝执行**（有权限限制）。
    
- **区别**：与 401 不同，你已经登录了，但你的权限不够（例如普通用户想进管理员后台）。
    
- **类比**：带了借书证，但试图闯入“馆长办公室”（禁止入内）。
    

### 4. `404 Not Found` (未找到)

- **含义**：服务器上找不到请求的资源。
    
- **误区**：**不是服务器挂了**，而是请求的路径（URL）不对或文件被删除了。
    

## 💥 5xx：服务器错误 (Server Error)

这一类状态码表示服务器在处理请求时内部发生了错误。

### 1. `500 Internal Server Error`

- **含义**：服务器遇到了不知道怎么处理的意外情况。
    
- **场景**：代码抛出异常、配置错误。
    

### 2. `503 Service Unavailable` (服务不可用)

- **含义**：服务器暂时无法处理请求（超载或维护）。
    
- **特征**：通常是**暂时性**的，稍后再试可能就恢复了。
    

### 3. `502 Bad Gateway` (网关错误)

- **角色**：出现在代理服务器/网关（如 Nginx）与后端应用服务器（如 Java/Python）之间。
    
- **含义**：网关从上游服务器接收到了**无效的响应**。
    
- **场景**：后端服务启动失败、响应格式错误。
    

### 4. `504 Gateway Timeout` (网关超时)

- **含义**：网关在规定时间内**没有收到**上游服务器的响应（超时）。
    
- **场景**：后端数据库查询太慢、死锁、处理时间过长。
    
- **辨析**：
    
    - **502** = 后端回话了，但回的是错的/乱码。
        
    - **504** = 后端一直不回话，网关等得不耐烦了。
