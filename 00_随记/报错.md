说明：
- 这是 ROS 2 的 rclpy 在查询节点列表时收到 RMW 层返回的空节点名导致的报错（rcl/graph.c:360）。
- 常见原因是系统里存在异常/非法命名的节点，或某个进程用错误的节点名注册到图中，导致 `ros2 node list` 解析失败。
- 该问题不是 ros2 CLI 本身的 Python 逻辑错误，而是底层 RMW/节点注册数据异常引起的。

补充（你当前场景）：
- 两台机子一边 Humble、一边 Jazzy，且都用 CycloneDDS（cyc）。跨发行版本身一般不会直接导致“空节点名”，但两边有任一节点发布了非法/异常节点名，仍会触发该错误。

定位思路（你描述为：MoveIt 一启动，VR 端就报错）：
- 先做隔离验证：临时把 VR 端或 MoveIt 端的 `ROS_DOMAIN_ID` 改成不同值，再启动 MoveIt，若错误消失，说明是跨机 DDS 数据触发。
- 在 MoveIt 端本机执行 `ros2 node list`，看是否也出现同样报错，确认异常是否来自 MoveIt 侧。
- MoveIt 端启动时开启更详细日志（如加 `--ros-args --log-level debug`），观察是否出现空/非法节点名相关日志。
- 检查 MoveIt 相关的 launch/参数里是否有 `name` 为空或包含非法字符的配置。

节点名合法性（快速判定）：
- 不能为空；不能只含 `/`。
- 只允许字母、数字、`_`、`/`；不能包含空格、`-`、`.` 等其它字符。
- 不能以数字开头；不能出现连续的 `/`。
