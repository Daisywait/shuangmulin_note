   HTTP 请求报文通常包含下面四大板块的内容：请求行、请求头、空行和请求体。

1. 请求行
   通常包括以下三部分：
   (a) 方法（Method）：常用的包括 GET、POST、PUT、DELETE 等
   (b) 资源路径：标识你要访问的服务器上的具体资源
   (c) HTTP 协议版本：如 HTTP/1.1 或 HTTP/2.0

2. 请求头
   以键值对的形式存储元数据，常见的包含以下九个字段：
   (a) Host：要访问的服务器域名
   (b) Accept：浏览器客户端能够接收的数据类型
   (c) Accept-Encoding：能够接收的编码类型
   (d) Authorization：登录凭证，通常是生成的 Token 令牌
   (e) Content-Type：请求体的媒体类型
   (f) Content-Length：请求体的内容长度
   (g) Cookie：浏览器本地存储的数据，通常在连接后由服务器发送给客户端
   (h) If-None-Match：资源的 ETag 值，用于缓存控制
   (i) Connection：连接选项，通常设置为 Keep-Alive 以保持长连接

3. 空行
   用于识别并分隔请求头与请求体，是一种固定的内容格式。

4. 请求体
   包含发送给服务器的具体数据，其格式由 Content-Type 指定。

响应报文同样包括状态行、响应头、空行和响应体。

1. 状态行
   (a) HTTP 协议版本
   (b) 状态码：例如 200 代表 OK，301 和 302 代表重定向
   (c) 状态消息：对状态码的简短文字描述

2. 响应头
   包含服务器回传的额外信息，常见的字段包括：
   (a) Content-Type：响应体的媒体类型
   (b) Content-Length：响应体的长度（字节数）
   (c) Server：服务器信息
   (d) Last-Modified：资源上次修改的时间
   (e) ETag： 响应体的实体标签，用于缓存和条件请求。，用于缓存控制
   (f) Expires：资源的过期时间
   (g) CORS 策略：跨源资源共享策略，指示哪些域可以访问资源
   (h) Location：用于重定向时的路径跳转
   (i) Set-Cookie：服务器发送给客户端并要求存储的 Cookie 数据

3. 响应体
   包含服务器返回给客户端浏览器的具体内容，如网页、文本、图片、视频等，也可能为空。
